线程池

线程池（Thread Pool）是一种优化多线程处理的设计模式，它通过预先创建和管理一组线程来处理任务，而不是每次有任务时都创建新的线程。这种机制能够有效提高系统的性能和资源利用率，特别是在并发执行大量短小任务的场景中。

线程池的工作原理

线程池的核心思想是复用一组固定数量的线程来执行任务，从而避免了频繁创建和销毁线程带来的开销。其主要工作流程如下：

  1. 初始化线程池：在程序开始运行时，创建一组（或动态调整）预定义数量的线程，并保持这些线程处于等待状态，直到有任务分配给它们。
  2. 提交任务：当有新的任务需要执行时，不是创建一个新的线程，而是将任务提交给线程池。
  3. 线程执行任务：线程池中的空闲线程会从任务队列中取出任务，开始执行任务。
  4. 线程复用：任务执行完毕后，线程不会被销毁，而是继续处于等待状态，准备执行下一个任务。
  5. 关闭线程池：当所有任务处理完毕，并且不再需要执行新的任务时，线程池可以被关闭，释放系统资源。

线程池的优点

  1. 减少线程创建销毁的开销：创建和销毁线程是耗费资源的操作，尤其在高并发的环境中，频繁的线程管理会影响性能。线程池通过复用线程，避免了这个问题。
  2. 更好的资源管理：通过控制线程池的大小，可以有效限制并发线程的数量，避免系统资源（如CPU、内存）被过多的线程占用，导致性能下降或崩溃。
  3. 提高任务执行效率：线程池通过管理任务队列和线程复用，能够快速响应和处理任务，减少任务等待的时间。
  4. 防止线程过多：线程池可以限制最大线程数量，避免因为创建过多线程导致系统崩溃或资源耗尽。

线程池的典型参数

  1. 核心线程数（core pool size）：线程池保持的最小线程数，即使这些线程处于空闲状态。
  2. 最大线程数（maximum pool size）：线程池允许的最大线程数量。当任务负载过大时，线程池可以增加到此数量。
  3. 任务队列：用于存放等待执行的任务。当所有线程都在忙碌时，新任务会被放入任务队列中，等待空闲线程来处理。
  4. 空闲线程存活时间：当线程池中的线程数量超过核心线程数时，空闲线程在超过指定的空闲时间后会被销毁，减少资源浪费。
  5. 拒绝策略：当任务队列已满且线程数已达到最大时，线程池的拒绝策略决定如何处理新任务（比如抛出异常、丢弃任务等）。

使用场景

  1. 服务器请求处理：服务器通常要处理大量并发请求，线程池能有效提高处理能力，避免为每个请求创建一个新线程。
  2. 并行计算：在需要并行处理多个计算任务时，线程池可以控制线程数量，保证计算资源的合理利用。
  3. 定时任务：线程池可以用于执行周期性任务或者延迟执行的任务。
